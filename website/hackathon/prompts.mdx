# 📚 Prompt Library

Stuck on what to type into Kiro? Use these as a starting point. Each theme below has:

- **Starter prompt** — your first feature.
- **Iteration prompts** — what to add next.
- **Stretch prompt** — if your team is flying.

:::note These are inspiration, not the official issue list
On the day, your facilitator will share the actual list of social and environmental issues your team can pick from. The themes below are example shapes — find the closest one and **adapt it to your problem and your persona** (see [Adapt to your problem](#-adapt-to-your-problem) below).
:::

## 🌊 Ocean and Water Pollution

Help communities report and track pollution in New Zealand's oceans, rivers, and beaches.

### Starter Prompt

```text
Create a web app where users can report water pollution at a beach or river.
The form should have fields for the location name, type of pollution (plastic,
sewage, chemical, or other), a text description of what they saw, and the date
they spotted it. Show all reports in a list below the form.
```

### Iteration Prompts

```text
Add a filter so users can view reports by pollution type. Show the count of
reports for each type at the top of the page.
```

```text
Add a "severity" rating to each report — low, medium, or high. Use colour-coded
badges (green, orange, red) to make it easy to scan the list.
```

```text
Add a feature that lets users upload a photo with their pollution report. Show
a thumbnail of the photo next to each report in the list.
```

### Stretch Prompt

```text
Add a dashboard page that shows a summary of all reports: total count, most
common pollution type, and a bar chart showing reports per month. Use the
existing report data to generate the chart.
```

## 💛 Youth Wellbeing

Build tools that support the mental health and wellbeing of young people in Aotearoa.

### Starter Prompt

```text
Create a daily mood tracker app. Users should be able to select how they're
feeling today from five options (great, good, okay, not great, rough) using
emoji buttons. Save each entry with the date and display a list of past
entries showing the emoji and date.
```

### Iteration Prompts

```text
Add an optional "journal note" text field so users can write a short note
about their day when they log their mood. Show the note alongside each
mood entry in the history list.
```

```text
Add a "weekly summary" section that shows how many times each mood was
selected in the last 7 days. Display it as a simple count like
"Great: 2, Good: 3, Okay: 1, Not great: 1, Rough: 0".
```

### Stretch Prompt

```text
Add a "self-care suggestions" feature. After a user logs their mood, show
a short encouraging message and one self-care tip based on their mood. For
example, if they select "rough", suggest "Try stepping outside for 5 minutes
of fresh air" or "Text a friend you trust and say hi."
```

## 🍲 Community Food Sharing

Help reduce food waste and food insecurity by connecting people who have spare food with those who need it.

### Starter Prompt

```text
Create a community food sharing board. Users can post a food listing with
the food name, a short description, the suburb or area for pickup, and an
expiry date. Show all active listings in a grid of cards. Each card should
display the food name, area, and how many hours until it expires.
```

### Iteration Prompts

```text
Add a search bar that filters the food listings by name or area as the user
types. Show a "no results" message if nothing matches.
```

```text
Add a "claim" button on each food card. When someone clicks it, mark that
listing as "claimed" and change the card's appearance so it looks different
from available listings. Claimed items should still be visible but greyed out.
```

```text
Add categories for the food listings: fruit and veg, baked goods, cooked meals,
pantry items, and other. Let users filter the board by category using buttons
at the top of the page.
```

### Stretch Prompt

```text
Add a simple stats section at the top of the page showing: total listings
posted, total items claimed, and the most active suburb. Update these numbers
automatically as users add and claim listings.
```

## 🦎 Native Species Conservation

Help protect Aotearoa New Zealand's unique native plants and animals through tracking, education, and community action.

### Starter Prompt

```text
Create a native species sighting tracker. Users can log a sighting by
selecting a species from a dropdown list (kiwi, kākāpō, tūī, kererū,
wētā, gecko, or other), entering the location name, adding a short note,
and picking the date. Display all sightings in a list sorted by most recent.
```

### Iteration Prompts

```text
Add a "species profile" section. When a user selects a species from the
dropdown, show a short info card with the species name, conservation status
(e.g. "Nationally Vulnerable"), and one fun fact. Use this data for at least
four species.
```

```text
Add a count of total sightings per species and display it as a simple
leaderboard at the top of the page. Show which species has been spotted
the most.
```

### Stretch Prompt

```text
Add a "conservation tips" page that lists five practical things young people
can do to help native species in their area — like building a wētā hotel,
planting native trees, or keeping cats indoors at night. Each tip should have
a title, a short description, and a difficulty rating (easy, medium, hard).
```

## 🏫 School Sustainability

Help schools track and reduce their environmental impact through waste audits, energy monitoring, and sustainability challenges.

### Starter Prompt

```text
Create a school waste audit tool. Users can log a waste audit entry by
selecting the waste type (landfill, recycling, compost, or e-waste),
entering the estimated weight in kilograms, and picking the date. Show
all entries in a table sorted by date with columns for waste type, weight,
and date.
```

### Iteration Prompts

```text
Add a summary section above the table that shows the total weight for each
waste type. Highlight the waste type with the highest total so the school
can see where to focus their efforts.
```

```text
Add a "sustainability challenge" feature. Display a weekly challenge at the
top of the page like "This week: try to reduce landfill waste by 10%."
Include a progress indicator that compares this week's landfill total to
last week's.
```

```text
Add the ability to record which class submitted each waste audit entry.
Include a "class" dropdown field in the form and add a leaderboard showing
which class has logged the most compost by weight.
```

### Stretch Prompt

```text
Add a "sustainability score" for the school. Calculate it based on the
percentage of total waste that goes to compost or recycling instead of
landfill. Display the score as a percentage with a colour indicator —
green for above 60%, orange for 30–60%, and red for below 30%. Show a
trend arrow indicating if the score is improving or declining compared
to last week.
```

## 🪄 Adapt to Your Problem {#-adapt-to-your-problem}

These prompts are templates, not scripts. The best results come when you rewrite them to fit **your specific problem and your persona**.

For example, take the Ocean Pollution starter prompt and adapt it for Mere (14, wants to know if her local beach is safe to swim at):

> *"Build a single-page web app for a 14-year-old who walks to the beach with friends on weekends. Show a list of 5 Auckland beaches with a 'Safe to swim?' status for each one. Add a button to report pollution at a beach. Use bright, friendly colours and big text."*

Same idea, but it's *your* idea now. Always:

- **Name your persona** in the prompt — *"Build for a teenager who…"*
- **Name the place** — *"Auckland beaches"*, *"a Hamilton high school"*
- **Pick one feature** — don't ask for five things at once

## ✨ Quick Prompt Tips

A vague prompt gets a vague result. **Be specific.** A good prompt tells Kiro three things:

1. **What** you're building (a form, a list, a map, a chart?)
2. **What data** it needs (which fields?)
3. **Where** it goes (new page, below the header, sidebar?)

| ❌ Vague | ✅ Specific |
|---|---|
| "Make a page about pollution" | "Add a page that lists 5 Auckland beaches with a 'Safe to swim?' status for each one" |
| "Add a map" | "Add a map showing a pin for each beach using its latitude and longitude" |
| "Fix the form" | "The date field doesn't save when I submit — fix the date picker" |
| "Make it look better" | "Use ocean blue as the page background and make the headings white and larger" |

:::tip Iterate, don't overthink
Your first prompt doesn't have to be perfect. Start small, see what Kiro builds, then refine. That's how real developers work too.
:::
