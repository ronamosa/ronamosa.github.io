# Project Structure

```text
.
├── website/                      # Docusaurus site root — all commands run here
│   ├── docusaurus.config.js      # Main site configuration
│   ├── sidebars.js               # Docs sidebar (autogenerated from filesystem)
│   ├── sidebars-changelog.js     # Changelog sidebar (autogenerated)
│   ├── package.json              # Dependencies and scripts
│   ├── blog/                     # Blog posts (Analysis & Essays)
│   ├── docs/                     # Technical documentation (engineer/, hacker/, study/, books/, archive/)
│   ├── changelog/                # Site changelog (second docs plugin instance, by year/month)
│   ├── src/
│   │   ├── pages/                # Custom React pages (index, about, projects, subscribe)
│   │   ├── components/           # Reusable React components
│   │   ├── theme/                # Swizzled Docusaurus theme overrides
│   │   ├── clientModules/        # GA4 event trackers
│   │   ├── css/custom.css        # Global style overrides
│   │   └── data/                 # Structured data (_Projects.js, _SocialLinks.js)
│   ├── plugins/                  # Custom Docusaurus plugins
│   └── static/                   # Static assets (img/, js/, pdf/, CNAME, robots.txt)
├── scripts/                      # Utility scripts (changelog helper, frontmatter fixers)
└── package.json                  # Root package (Algolia search dependency)
```

## Dual Docs Plugin Architecture

The site runs two instances of `@docusaurus/plugin-content-docs`:

1. **Main docs** (via preset-classic) — serves `website/docs/` at `/docs/`
2. **Changelog** (standalone plugin, id: `changelog`) — serves `website/changelog/` at `/changelog/`

Both use autogenerated sidebars. New sections follow the same pattern: add a content directory, register a plugin instance in `docusaurus.config.js`, and create a matching sidebar config.

## Adding a New Section

- Content directory goes under `website/` (sibling to `docs/`, `blog/`, `changelog/`)
- Register as a new `@docusaurus/plugin-content-docs` instance in `docusaurus.config.js`
- Create a sidebar file (`sidebars-{section}.js`) with autogenerated config
- Use `_category_.json` files in subdirectories to control sidebar labels and ordering
- Add a navbar item in `docusaurus.config.js` `themeConfig.navbar.items`

## File Naming Conventions

- **Blog**: `YYYY-MM-DD - Title-With-Dashes.md` or `YYYY-MM-DD-kebab-title.md`
- **Docs**: `kebab-case-name.md`
- **Changelog**: `website/changelog/{YEAR}/{month}.md` — entries at the TOP (most recent first)
